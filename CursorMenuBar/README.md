# CursorMenuBar source layout

This target is split by responsibility so it stays maintainable without “vibe coding” and reflects current Swift/SwiftUI practice.

## Structure

| File | Responsibility |
|------|----------------|
| **CursorMenuBarApp.swift** | App entry point, `AppDelegate`, status bar + floating panel, `AppState`, `FloatingPanel`. |
| **PopoutView.swift** | Main popout UI: layout, composer dock, send/stream orchestration, streaming state updates. ~550 lines. |
| **ConversationModels.swift** | Domain types: `ConversationSegment`, `ConversationTurn`, `StreamPhase`, tool-call enums/structs. |
| **AgentTabState.swift** | Tab state: `AgentTab`, `TabManager` (observable objects). |
| **CursorTheme.swift** | Design tokens (colors, gradients) and app constants: `CursorTheme`, `AvailableModels`, `QuickActionPrompts`, `AppLimits`. |
| **WorkspaceHelpers.swift** | Workspace/git and prompt helpers: `loadDevFolders`, `loadGitBranches`, `gitCheckout`, `screenshotPaths`, `estimatedContextTokens`, `autoGeneratedTabTitle`, `nextScreenshotPath`. |
| **AgentRunner.swift** | Cursor CLI integration: `createChat()`, `stream()`, stream parsing, `AgentStreamChunk`, `AgentToolCallUpdate`, `AgentRunnerError`. |
| **SubmittableTextEditor.swift** | Composer input: `PasteAwareTextView` (AppKit), `SubmittableTextEditor` (SwiftUI wrapper), paste-image handling. |
| **BrandMark.swift** | Brand logo / header view. |
| **TabChip.swift** | Tab bar chip view. |
| **OutputScrollView.swift** | Scrollable output area with scroll-to-bottom pill. |
| **ConversationViews.swift** | Conversation UI: `visibleSegments`, `ConversationSegmentView`, `ConversationTurnView`, `ProcessingPlaceholderView`, text normalization. |
| **SettingsView.swift** | Settings window (workspace path). |

## Practices used

- **Single responsibility**: Each file has a clear role (models, theme, helpers, one main view, small view components).
- **Shared types in one place**: Conversation and tool-call types live in `ConversationModels`; theme/constants in `CursorTheme`.
- **No giant view file**: `PopoutView` composes smaller views (`BrandMark`, `TabChip`, `OutputScrollView`, `ConversationTurnView`, `SubmittableTextEditor`) and focuses on layout and streaming logic.
- **MARK comments**: Used in longer files to separate sections.
- **Internal visibility**: Everything is internal (default); no `public` unless the target is used as a module.
