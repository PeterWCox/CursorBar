Cursor Metro Onboarding Concept

Design Direction

A calmer first-run state with one clear decision at a time. The existing Projects hub stays available after setup, but the first impression becomes guided instead of operational.

Intent

Show only on clean launch
Return until setup succeeds
Reuse existing project flows

Outcome

New users land in a focused welcome screen. Experienced users still get the familiar project controls once onboarding is complete.

What changes in the app

The wizard appears only for fresh local state, then disappears permanently after the first successful setup.

1
Fresh launch detected No saved state, no configured roots, no restored manual projects.
2
User chooses one path Open, create, or clone. Scan folders stays as a secondary option.
3
Success marks onboarding complete Later launches fall back to the regular Projects hub instead of the wizard.
